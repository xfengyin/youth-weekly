### 🤖 AI 编程助手横评

主流 AI 编程助手对比：

| 工具 | 优点 | 缺点 | 适合人群 |
|------|------|------|---------|
| GitHub Copilot | 生态好、响应快 | 订阅制 | 全栈开发者 |
| Cursor | AI 功能丰富 | 生态较弱 | 追求创新 |
| Claude Code | 上下文理解强 | 中文支持一般 | 后端开发者 |
| 通义灵码 | 中文友好 | 功能较少 | 国内用户 |
| 文心快码 | 集成百度生态 | 生态有限 | 百度用户 |

**使用建议：**
- 可以同时使用多个工具，取长补短
- 重要代码一定要 review
- AI 是助手，不是替代品

### 📝 实用 GitHub 技巧

**1. 键盘快捷键**
- `T` - 文件搜索
- `S` - 全局搜索
- `W` - 切换分支
- `Shift + ?` - 显示所有快捷键

**2. GitHub Actions 技巧**
```yaml
# 缓存依赖加速 CI
- uses: actions/cache@v4
  with:
    path: node_modules
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
```

**3. GitHub CLI 常用命令**
```bash
# 快速克隆并进入目录
gh repo clone owner/repo && cd repo

# 创建 PR
gh pr create --title "feat: 新功能" --body "描述"

# 查看 PR 状态
gh pr status
```

### 💼 技术面试完全指南

金三银四已过，但面试技巧永不过时：

**面试流程：**
1. HR 筛选
2. 技术初面（基础）
3. 技术复面（项目）
4. 主管面（综合）
5. HR 谈薪

**常见问题准备：**

| 问题类型 | 准备要点 |
|---------|---------|
| 自我介绍 | 2 分钟，突出亮点 |
| 项目介绍 | STAR 法则，量化成果 |
| 技术问题 | 原理 + 实践 |
| 反问环节 | 问团队、技术栈、发展 |

**薪资谈判技巧：**
- 先了解市场行情
- 给自己设定底线
- 不要先开口说数字
- 谈整体包而非单一项

### 🖥️ 开发工具

#### 1. GitHub Copilot 进阶用法

Copilot 不只是代码补全：

**Prompt 工程：**
```
# 在注释中描述需求
// 生成一个防抖函数，延迟 300ms，支持取消

# 指定代码风格
// 使用 TypeScript，箭头函数，纯函数风格

# 生成测试用例
// 为上面的函数生成 5 个测试用例
```

**Chat 功能：**
- `/explain` - 解释选中代码
- `/fix` - 修复错误
- `/test` - 生成测试
- `/refactor` - 重构代码

---

#### 2. 终端工具链

打造高效终端：

**推荐组合：**
- **终端**：Warp / Alacritty
- **Shell**：Zsh + Oh My Zsh
- **主题**：Powerlevel10k
- **补全**：zsh-completions

**实用配置：**
```bash
# ~/.zshrc 优化
alias gs='git status'
alias gd='git diff'
alias gc='git commit'
alias gp='git push'

# 效率工具
alias k='kubectl'
alias d='docker'
alias dk='docker compose'
```
